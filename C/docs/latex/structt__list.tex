\doxysubsection{Detailed Description}
Lista simplesmente lincada. Está longe de ser uma biblioteca adequada para essa estrutura de dados. 

Foi implementado apenas o suficiente para implementar a \href{https://github.com/y-azvd/calculator/tree/master/C}{\texttt{ calculadora em C}}

Na documentação, as funções para lista estão na seção \char`\"{}\+Membros públicos\char`\"{}. Apesar de C não ser uma linguagem que tem suporte para orienteção objeto, as funções foram indicadas com "{\ttfamily @memberof \mbox{\hyperlink{structt__list}{t\+\_\+list}}}" para juntar a definição da estrutura de dados \mbox{\hyperlink{structt__list}{t\+\_\+list}} e suas funções na mesma página, além de dar a ideia de uma classe e seus métodos.

\begin{DoxyWarning}{Warning}
Cabe ressaltar ao usuário que a lista implementada \char`\"{}não sabe\char`\"{} se o espaço de memória apontado {\ttfamily node-\/$>$data} foi alocado dinamicamente ou não. Essa resposabilidade é deixada ao usuário (até porque não sei outra maneira de fazer isso). Esse aviso é ressaltado ao longo da documentação em algumas funções (\mbox{\hyperlink{structt__list_a002043e4f5f58844e2fafa135691a2e2}{remove\+\_\+head()}}, \mbox{\hyperlink{structt__list_a2856474aa1a64f60a649cc55d84e5633}{delete\+\_\+head()}}, \mbox{\hyperlink{structt__list_ac3c7cffd8032adc9fbe65be10d8a7e61}{soft\+\_\+pop()}}). Resumidamente, essa biblioteca libera a memória apontada pelo nó, mas a memória apontada por {\ttfamily void$\ast$ data} (em \mbox{\hyperlink{structt__node}{t\+\_\+node}}) deve ser analisada em cada caso, dependendo da aplicação do usuário, que deve escolher qual função é mais apropriada.
\end{DoxyWarning}
Essa lista foi testada usando \href{http://www.valgrind.org/}{\texttt{ valgrind}} e a framework \href{https://github.com/catchorg/Catch2\#whats-the-catch}{\texttt{ Catch2}}. Alguns trechos dos arquivos testes são usados como exemplo, por isso algumas macros do Catch2 estão presentes. Se você não tem familiriadade, cheque a parte \href{https://github.com/catchorg/Catch2/blob/master/docs/tutorial.md\#test-cases-and-sections}{\texttt{ Testes e Seções}}.
